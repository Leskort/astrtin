# Варианты хранения фотографий (до 200 штук)

Для небольшого количества фотографий (до 200 штук) есть несколько вариантов:

## Вариант 1: Supabase Storage (РЕКОМЕНДУЕТСЯ) ⭐

**Преимущества:**
- ✅ Бесплатный план: 1GB хранилища (хватит на ~200 фотографий)
- ✅ Работает во многих странах
- ✅ Быстрая загрузка
- ✅ Простая настройка

**Настройка:**
См. файл `SUPABASE_SETUP.md`

**Лимиты:**
- 1GB бесплатно (примерно 200 фотографий по 5MB каждая)
- Если нужно больше - можно перейти на платный план

---

## Вариант 2: Cloudinary

**Преимущества:**
- ✅ Бесплатный план: 25GB хранилища
- ✅ Автоматическая оптимизация изображений
- ✅ CDN для быстрой загрузки

**Недостатки:**
- ❌ Может быть недоступен в некоторых странах

**Настройка:**
См. файл `CLOUDINARY_SETUP.md`

---

## Вариант 3: Firebase Storage

**Преимущества:**
- ✅ Бесплатный план: 5GB хранилища
- ✅ Хорошая интеграция с другими сервисами Google

**Настройка:**
1. Создайте проект на https://firebase.google.com
2. Включите Storage
3. Получите ключи API
4. Добавьте переменные в Netlify:
   - `FIREBASE_PROJECT_ID`
   - `FIREBASE_CLIENT_EMAIL`
   - `FIREBASE_PRIVATE_KEY`

---

## Вариант 4: Локальное хранилище (только для разработки)

**Использование:**
- Работает только в development режиме
- Фотографии сохраняются в `public/uploads/`
- Не работает на Netlify (файловая система только для чтения)

---

## Рекомендация для 200 фотографий

Для вашего случая (максимум 200 фотографий) **рекомендую Supabase**:

1. **Бесплатно** - 1GB хватит на 200 фотографий
2. **Простая настройка** - всего 2 переменные окружения
3. **Работает везде** - нет географических ограничений
4. **Быстро** - хорошая скорость загрузки

### Расчет места:
- 200 фотографий × 5MB = 1GB (вписывается в бесплатный план)
- Если фотографии меньше - поместится еще больше
- Если больше - можно использовать оптимизацию изображений

---

## Быстрая настройка Supabase

1. Зайдите на https://supabase.com
2. Создайте проект (бесплатно)
3. Создайте Storage bucket `astrinn-photos` (публичный)
4. Получите ключи из Settings → API
5. Добавьте в Netlify:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
6. Пересоберите проект

Подробная инструкция: `SUPABASE_SETUP.md`

---

## Что делать если превысите лимит?

Если у вас будет больше 200 фотографий:

1. **Удаляйте старые** - приложение поддерживает удаление
2. **Оптимизируйте размер** - сжимайте фотографии перед загрузкой
3. **Перейдите на платный план** - Supabase Pro стоит $25/месяц (100GB)

---

## Текущая конфигурация

Приложение автоматически выбирает хранилище в следующем порядке:
1. Supabase (если настроен)
2. Cloudinary (если настроен)
3. Локальное хранилище (только development)

Просто настройте один из вариантов выше, и все будет работать автоматически!
