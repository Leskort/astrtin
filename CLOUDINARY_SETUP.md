# Настройка Cloudinary для загрузки фотографий

## Шаг 1: Создайте аккаунт Cloudinary

1. Перейдите на https://cloudinary.com
2. Зарегистрируйтесь (бесплатный план включает 25GB хранилища)
3. После регистрации вы попадете в Dashboard

## Шаг 2: Получите учетные данные

В Dashboard Cloudinary найдите раздел "Account Details" или перейдите в Settings:

- **Cloud Name** - имя вашего облака (например: `demo`)
- **API Key** - ваш API ключ
- **API Secret** - ваш секретный ключ

## Шаг 3: Добавьте переменные окружения в Netlify

1. Откройте ваш проект в Netlify Dashboard
2. Перейдите в **Site settings** → **Environment variables**
3. Добавьте следующие переменные:

```
CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-api-key
CLOUDINARY_API_SECRET=your-api-secret
```

⚠️ **ВАЖНО**: Не добавляйте эти переменные в публичный репозиторий!

## Шаг 4: Пересоберите проект

После добавления переменных окружения:
1. Перейдите в **Deploys**
2. Нажмите **Trigger deploy** → **Deploy site**

Или просто сделайте новый commit и push в GitHub - Netlify автоматически пересоберет проект.

## Проверка

После настройки попробуйте загрузить фотографию:
- Если Cloudinary настроен правильно - фотография загрузится
- Если нет - вы увидите сообщение об ошибке с инструкцией

## Альтернатива: Использование без Cloudinary

Если вы не хотите использовать Cloudinary, код будет работать в development режиме с локальной файловой системой. Но для production на Netlify нужен облачный провайдер.

## Другие варианты

Вы также можете использовать:
- **Supabase Storage** - бесплатный план с 1GB
- **Firebase Storage** - бесплатный план с 5GB
- **AWS S3** - платный, но очень надежный

Для интеграции с другими провайдерами нужно обновить файл `lib/cloudinary.ts` и `app/api/photos/upload/route.ts`.
